journal_three(firefox) {

For Journal 3, I searched for a site that demonstrated Steve Krug's usability rules as presented in chapters 4 and 5. These chapters discussed the inclusion of useful links and bodies of text. I discovered the Access Firefox site, which I felt demonstrated Krug's recommendations well.

}

'Access Firefox's Purpose

Upon first glance, the Access Firefox homepage is a bit busy. It's purpose isn't immediately apparent, partly due to its visual hierarchy. The two largest elements on the front page are both links to download Firefox. This, alongside the name of the site ('Access Firefox') could lead some individuals to think that this website is simply meant as a hub for downloading the Firefox browser.

Fortunately, the next piece of the homepage that I scanned made the site's purpose clear. The subheading "To sum it up, we provide:" concisely identifies the site's purpose, which is to provide accessibility tools to Firefox users with disabilities.

After reading this introductory paragraph, I moused over the menu elements on the left-hand-side which provided a clean and easy-to-understand interface for navigating the site, and made it clear that I could use this site to download add-ons for Firefox.

Critique

Overall, Access Firefox was successful in providing its users with mindless choices. Both the links on the left-hand-side and the links in the main content column lead to the same pages. This redundancy accomodates the typical, impatient Internet user who probably visited this site for a specific purpose (to download an add-on).

One feature that I found to be distracting was the choice of icons for some of the links (see screenshot below). For example, the "Firefox Accessibility Features" link was paired with a set of keys. I believe many users, including myself, would associate keys with privacy features. This, coupled with the word "accessibility", may stop users for a brief moment and cause them to think.

For the most part, I believe Access Firefox followed Krug's third rule: omit words. The website contains very little, if any, "happy talk". While the website does have an introductory paragraph, it is purely functional and does not attempt to justify its existence or explain why it's better than any other website. Some pages (e.g. the "Firefox Accessibility Extensions" page) could use some pruning, but the author's technique of breaking his/her webpages into small paragraphs makes readability a near non-issue.

Potential Improvements

My first change would be to make the purpose of the site more apparent. I would probably provide a heading near the top, something like: "Accessibility Tools for Firefox Users with Visual Impairment". This would immediately communicate the site's purpose.

Next, I would remove the large icons on the home page, or change them to something more closely associated with the links they're paired with. Perhaps even placing the links in blue boxes would be an improvement.
